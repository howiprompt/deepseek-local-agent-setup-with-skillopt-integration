## The Architecture: Why This Stack Works

Most developers fail at local agents because of the "Context Switch Tax." Your inference engine runs in C++, your skills live in Python, and your interface is a hacked-together Node.js server. Every switch costs latency and introduces breaking points.

The **Local Agentic Stack** solves this through a unified IPC (Inter-Process Communication) layer. We use a shared memory bridge to allow the DS4 inference engine to push logits directly into the SkillOpt Python environment without JSON serialization overhead.

1.  **DS4 Core:** The raw, quantized inference engine.
2.  **SkillOpt:** The "brain" that edits trajectories. It accepts the DS4 output, evaluates it against a success criteria, and updates the agent's internal skill weights.
3.  **Odysseus:** The containerization and workspace manager. It handles the file system, the "75 skills" JSON pack, and the lifecycle of the agent.
4.  **The Dashboard:** A CLI interface built on `textual` that gives you real-time telemetry.

### Execution Logic
To launch the inference engine, the stack uses the `launcher.sh` script. It detects your OS and architecture and spins up the server.

```bash
#!/bin/bash
# launcher.sh - Auto-detects and launches DS4

ARCH=$(uname -m)
OS=$(uname -s)

if [[ "$OS" == "Darwin" ]]; then
    if [[ "$ARCH" == "arm64" ]]; then
        echo "[SYSTEM] Launching DS4 on Metal (Apple Silicon)..."
        ./binaries/ds4-metal-macos --config ds4_config.json
    else
        echo "[ERROR] Intel Macs are not supported for this stack."
        exit 1
    fi
elif [[ "$OS" == "Linux" ]]; then
    if lspci | grep -i nvidia > /dev/null; then
        echo "[SYSTEM] Launching DS4 on CUDA..."
        ./binaries/ds4-cuda-linux --config ds4_config.json
    elif lspci | grep -i amd > /dev/null; then
        echo "[SYSTEM] Launching DS4 on ROCm..."
        ./binaries/ds4-rocm-linux --config ds4_config.json
    else
        echo "[ERROR] No supported GPU detected. Falling back to CPU (not recommended)."
        ./binaries/ds4-cpu-linux --config ds4_config.json
    fi
fi
```

## Deliverable 2: Integrated SkillOpt Python Environment

This is the secret sauce. SkillOpt is not just a prompt wrapper; it performs "Trajectory-Driven Editing." It observes the chain-of-thought generated by DS4 and optimizes the *path* the agent takes to solve a problem.

### The SkillOpt Class
We bundle a pre-configured virtual environment (`.venv`) with the stack. Here is the core `SkillOptimizer` class included in the package.

import json
import zmq
import time
from typing import List, Dict

class SkillOptimizer:
    def __init__(self, inference_port: int = 5555):
        self.context = zmq.Context()
        self.socket = self.context.socket(zmq.REQ)
        self.socket.connect(f"tcp://localhost:{inference_port}")
        self.trajectory_log = []
        
    def query_ds4(self, prompt: str) -> str:
        """Send prompt to DS4 binary and return raw completion."""
        payload = {"prompt": prompt, "max_tokens": 512}
        self.socket.send_json(payload)
        response = self.socket.recv_json()
        return response.get("completion", "")
    
    def execute_skill(self, skill_name: str, params: Dict):
        """
        Executes a skill and records the result.

            "result": result,
            "reward": 0.0 # Placeholder for reinforcement signal

        return result

## Deliverable 3: JSON Skill-Pack (The '75 Skills' Format)

The agent needs a library of actions it can take. We have standardized these into the `html-anything` compatible format. This allows the agent to "read" web structures or file systems as HTML and interact with them.

### The Schema
The schema is designed for fast parsing by the DS4 model. It minimizes token usage on structural overhead.

```json
{
  "skill_id": "file_sys_scan",
  "description": "Recursively scans a directory and returns file metadata",
  "input_format": "path_string",
  "output_format": "json_list",
  "html_compatibility": true,
  "parameters": {
    "target_dir": { "type": "string", "required": true },
    "depth": { "type": "integer", "default": 3 }
  },
  "prompt_template": "Scan directory {target_dir} to depth {depth}. Return results as HTML list."
}
```

### The Pack (`skills_pack.json`)
We include 75 pre-defined skills in the `workspace/skills/` directory. Here is a preview of 3 critical ones included in the full pack to get you started immediately:

1.  **Web Scraper (`web_fetch_html`):** Connects to a URL, strips ads, returns content HTML.
2.  **Code Auditor (`code_security_scan`):** Runs basic regex checks on Python/C++ files for vulnerabilities.
3.  **Data Summarizer (`vector_chunk_summary`):** Breaks down large text files into vector-ready chunks.

### Integration with SkillOpt
When DS4 generates a thought like "I need to check the server logs," the SkillOpt parser matches this intent against the `skills_pack.json` using semantic embedding (handled locally).
